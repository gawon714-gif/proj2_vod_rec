# Report Agent — 개발 진행상황 보고서 작성

## 역할
user_embedding 개발 중간중간 진행상황을 보고서로 작성하여 reports 폴더에 저장.

## 실행 시점
- Phase 완료 시 (Phase 1, 2, 3, 4 각각)
- 주요 기능 구현 완료 시
- 사용자가 보고서 작성 요청 시

## 저장 위치
```
user_embedding/reports/YYYY-MM-DD_{작업명}.md
예) user_embedding/reports/2026-03-10_phase1_db_setup.md
```

## 보고서 필수 포함 항목

### 1. 작업 개요
- 작업한 Phase 및 목표
- 작업 기간

### 2. 완료된 작업
- 구현한 기능 목록
- 생성/수정된 파일

### 3. 테스트 결과
- 작성한 테스트 케이스 수
- 통과/실패 현황
- 커버리지

### 4. 진행상황 요약
| Phase | 상태 | 완료율 |
|-------|------|--------|
| Phase 1 | 완료/진행중/대기 | 0~100% |
| Phase 2 | ... | ... |
| Phase 3 | ... | ... |
| Phase 4 | ... | ... |

### 5. 이슈 및 해결
- 발생한 문제
- 해결 방법

### 6. 다음 단계
- 이후 진행할 작업

## 보고서 작성 규칙
- 핵심 수치는 표로 정리
- 테스트 결과 반드시 포함
- 노션 붙여넣기 가능한 마크다운 형식
- 민감 정보 (API 키, 비밀번호) 절대 포함 금지
