# Report Agent — 작업 레포트 작성

## 역할
주요 작업 완료 후 어떤 작업이 이루어졌는지 레포트를 자동으로 작성하고 저장하는 Agent.

## 실행 시점
- 주요 기능 개발 완료 시
- 데이터 처리 파이프라인 완료 시
- 사용자가 레포트 작성 요청 시

## 저장 위치
```
{브랜치명}/reports/YYYY-MM-DD_{작업명}.md
예) RAG/reports/2026-03-10_RAG_pipeline.md
```

## 레포트 필수 포함 항목

### 1. 작업 개요
- 작업 목적
- 작업 기간
- 담당 브랜치

### 2. 사용 도구 및 API
- 사용한 외부 API (TMDB, KMDB 등)
- 라이브러리/프레임워크
- DB 연결 정보 (민감 정보 제외)

### 3. 처리 결과
- 전/후 수치 비교
- 성공/실패 건수
- 소스별 분포

### 4. 데이터 정제 내역
- 어떤 이상치를 발견했는지
- 어떻게 처리했는지
- 처리 건수

### 5. 한계 및 미처리 항목
- 처리하지 못한 데이터
- 원인 분석
- 향후 보완 방법

### 6. 다음 단계
- 이후 진행할 작업
- 우선순위

## 레포트 작성 규칙
- 핵심 수치는 표로 정리
- 코드 스니펫은 코드블록 사용
- 노션 붙여넣기 가능한 마크다운 형식
- 민감 정보 (API 키, 비밀번호) 절대 포함 금지
